```
总耗时: 60+ 秒 (经常超时失败)

分解：
┌─────────────────────────────────────────────────────────┐

│ 原因: waitUntil: 'networkidle' 策略                    │

└─────────────────────────────────────────────────────────┘
```

## 🔍 详细的瓶颈分析

### 瓶颈 #1: 网络导航超时 🔴 **最严重**

```
当前代码 (controller.ts:331):
await page.goto(action.url, { waitUntil: 'networkidle' })

问题:
  networkidle = 等待所有网络活动停止
  
百度首页的网络请求特点:
  1. 主 HTML 加载: 500ms
  2. CSS/JS 加载: 1-2s
  3. 图片资源: 2-5s
  4. 第三方脚本: 5-10s
     - Google Analytics
     - 百度统计
     - 广告网络
  5. WebSocket 连接: 持续活动
  6. 后台数据加载: 无限期

结果:
  30s 后还有 HTTP 请求 → Playwright 给放弃等待 → 超时异常

影响范围:
  ❌ 100% 的网站导航都受影响
  ❌ 导致任务失败率 > 80%
```

**影响**: ⏱️ 30+ 秒/次 (可能多次重试)

---

优化前:
