发现 1️⃣  网络导航是主要瓶颈
┌─────────────────────────────────────────────────────────────────┐
│ 原因: waitUntil: 'networkidle' 策略                            │
│                                                                 │
│ 问题: 百度首页有持续的后台请求 (广告、分析、WebSocket)         │
│      → 30 秒还没停 → 超时异常                                   │
│                                                                 │
│ 影响: 100% 的导航任务都被卡住                                   │
│                                                                 │
│ 解决: 改用 'domcontentloaded' (等待 DOM 解析完成)               │
│      → 2-3 秒完成 → 页面已可交互                                │
└─────────────────────────────────────────────────────────────────┘
