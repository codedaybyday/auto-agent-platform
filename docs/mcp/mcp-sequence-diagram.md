## 角色职责

### Server (Node.js)

| 职责 | 说明 |
|------|------|
| **MCP Host** | 容纳和管理多个 MCP Client |
| **外部连接** | 连接 GitHub、Slack、PostgreSQL 等外部 MCP Server |
| **本地连接** | 通过 stdio 启动并连接 Client (MCP Server) |
| **工具发现** | 从所有连接的 MCP Server 获取工具列表 |
| **统一路由** | 根据工具名称路由到对应的 MCP Client |

### Client (Electron Main)

| 职责 | 说明 |
|------|------|
| **MCP Server** | 暴露本地工具能力 |
| **工具注册** | 注册 browser、bash、file 等本地工具 |
| **请求处理** | 处理 Server 发来的 `listTools` 和 `callTool` 请求 |
| **本地执行** | 使用本地资源执行具体操作 (CDP、child_process、fs) |

## 关键设计决策

### 1. 为什么用 stdio 而不是 WebSocket?

| 方案 | 优点 | 缺点 |
|------|------|------|
| **stdio** | 进程生命周期绑定、自动清理、防火墙友好 | 只能本地、无法远程 |
| **WebSocket** | 可远程、双向流 | 需要额外端口管理、进程解耦、需心跳保活 |
| **SSE** | 服务器推送友好 | 单向通信、需要额外处理调用 |

**决策**: 本地工具用 stdio，外部服务用 SSE

### 2. WebSocket 还保留吗?

保留，但用途改变:

| 用途 | 协议 |
|------|------|
| 工具调用 | ❌ WebSocket (改为 MCP stdio) |
| 流式消息 | ✅ WebSocket (SSE 格式) |
| 状态同步 | ✅ WebSocket |
| 人工确认 | ✅ WebSocket |

### 3. 工具命名冲突?

```typescript
// 解决方案: 命名空间前缀
// 外部工具
"github.get_issues"
"slack.send_message"
"postgres.query"

// 本地工具
"local.browser_navigate"
"local.bash"
"local.file_read"
```

## 消息格式对比

### 旧方案 (WebSocket 自定义)
```json
{
  "type": "tool.execute",
  "messageId": "123",
  "timestamp": 1234567890,
  "sessionId": "xxx",
  "payload": {
    "toolCall": {
      "id": "call_1",
      "name": "browser_navigate",
      "arguments": {"url": "baidu.com"}
    },
    "timeout": 60000
  }
}
```

### 新方案 (MCP stdio)
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "browser_navigate",
    "arguments": {
      "url": "baidu.com"
    }
  }
}
```

## 时序总结

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1-2 | 启动 Client 进程 | Server 通过 stdio 启动 Client MCP Server |
| 3-4 | MCP 协议握手 | 确认版本和能力 |
| 5-6 | 工具发现 | Server 获取 Client 工具列表 |
| 7 | 存储映射 | Server 维护工具到 MCP Client 的映射 |
| 8-10 | 用户请求 | 用户发送消息，启动 Agent Loop |
| 11-13 | 准备上下文 | 获取所有工具，发送给 LLM |
| 14 | LLM 决策 | LLM 决定调用哪个工具 |
| 15-17 | 工具路由 | Server 路由到对应的 MCP Client |
| 18 | 本地执行 | Client 执行具体操作 |
| 19-22 | 返回结果 | 层层返回，LLM 生成回复 |
