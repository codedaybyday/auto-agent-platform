# 测试场景设计规范

本文档定义各类前端模块的测试场景设计规范，供 `unit-test-generator` skill 使用。

---

## 一、React 组件测试场景

### 1.1 渲染测试
- 默认 props 下正常渲染，快照不变
- 必填 props 缺失时的降级渲染（空态、loading 态）
- 不同 props 组合下的条件渲染（v-if / 三元表达式控制的 UI 块）
- 列表渲染：空数组、单条、多条、超长列表

### 1.2 交互测试
- 点击事件：触发回调、更新状态、跳转路由
- 表单输入：onChange 触发、受控/非受控值更新
- 表单提交：校验通过提交、校验失败阻止提交
- 防抖/节流：快速连续触发只执行一次

### 1.3 异步行为测试
- 组件挂载后自动发起请求（useEffect）
- 请求成功：数据渲染到 DOM
- 请求失败：错误提示渲染
- 请求中：loading 状态渲染
- 组件卸载后请求返回：不触发 setState（避免内存泄漏警告）

### 1.4 状态管理测试
- 初始状态正确
- dispatch action 后状态更新
- 多个组件共享同一 store 状态

## 二、Vue 组件测试场景

### 2.1 渲染测试
- 默认 props 渲染
- 插槽内容渲染（default slot、具名 slot）
- v-if / v-show 条件渲染
- v-for 列表渲染

### 2.2 交互测试
- 点击事件触发 emit
- 双向绑定（v-model）值更新
- 表单提交

### 2.3 异步行为测试
- onMounted 中的异步请求
- 请求成功/失败的 UI 响应
- nextTick 后的 DOM 更新

---

## 三、Hooks / Composables 测试场景

### 3.1 初始状态
- 返回值的初始状态符合预期
- 初始化时不触发副作用（除非设计如此）

### 3.2 状态变更
- 调用返回的方法后，状态正确更新
- 多次调用的幂等性（如果应该幂等）
- 依赖参数变化时，状态重新计算

### 3.3 异步 Hooks
- 请求发起前的 loading 状态
- 请求成功后的数据状态
- 请求失败后的 error 状态
- 重复调用时取消上一次请求（如有防抖/竞态处理）

### 3.4 副作用清理
- 组件卸载时，定时器/事件监听被清除
- 组件卸载后，异步回调不再执行

---

## 四、工具函数测试场景

### 4.1 正常输入
- 典型输入 → 预期输出
- 多组不同输入的参数化测试（`it.each`）

### 4.2 边界值
| 类型 | 边界值 |
|------|--------|
| 数字 | 0、负数、最大安全整数、NaN、Infinity |
| 字符串 | 空字符串、单字符、超长字符串、特殊字符（`<>&"'`）、Unicode |
| 数组 | 空数组、单元素、大数组 |
| 对象 | 空对象、嵌套对象、循环引用（如适用） |
| 日期 | 无效日期、跨时区、闰年 |

### 4.3 异常输入
- null / undefined 参数
- 错误类型参数（传字符串给期望数字的参数）
- 应抛出错误的场景：验证 `expect(() => fn()).toThrow()`

---

## 五、API 请求层测试场景

### 5.1 请求构造
- 请求参数正确传递（URL、headers、body）
- 参数序列化格式正确

### 5.2 响应处理
- 成功响应（200）：数据正确解析
- 业务错误（code !== 0）：错误信息正确提取
- HTTP 错误（4xx/5xx）：错误正确抛出
- 网络超时：超时错误正确处理

### 5.3 拦截器
- 请求拦截器：token 注入、参数处理
- 响应拦截器：统一错误处理、数据格式转换

---

## 六、Store（状态管理）测试场景

### 6.1 初始状态
- store 初始化后各字段值符合预期

### 6.2 Actions / Mutations
- 同步 action：状态正确更新
- 异步 action：请求成功后状态更新，请求失败后错误状态更新
- 多个 action 的执行顺序

### 6.3 Getters / Computed
- 基于 state 的派生值计算正确
- state 变化后 getter 重新计算

---

## 七、场景优先级

当被测模块复杂、场景众多时，按以下优先级生成：

1. **P0（必须）**：主流程 Happy Path + 核心异常（API 失败、权限不足）
2. **P1（重要）**：边界条件 + 常见异常（空值、类型错误）
3. **P2（补充）**：罕见边界 + 性能相关（大数据量、并发）

如果技术方案中标注了「高风险」或「核心逻辑」的模块，该模块的所有场景都升级为 P0。
