# PR 文档模板

生成 PR 时，使用以下模板。将 `[占位符]` 替换为实际内容。

---

```markdown
# PR: [skill-name] — [一句话描述本次修改的目的]

**创建时间**: [YYYY-MM-DD HH:MM]
**修改的 Skill**: `[skill-name]`
**Skill 路径**: `~/.catpaw/skills/[skill-name]/SKILL.md`
**触发原因**: [失败案例来源，如"PRD 评审被打回 3 次"、"测试报告显示边界条件处理错误"]
**Patch 数量**: [N] 个
**影响范围**: [轻微 / 中等 / 重大]

---

## 背景

[2-3 句话描述问题背景：AI 在什么场景下失败了，失败了多少次，影响是什么。
例如：过去两周内，PRD 生成 skill 产出的文档被产品评审打回 3 次。
主要问题集中在两个方面：缺少竞品分析章节，以及用户故事格式不符合公司规范。
这导致每次都需要人工补充，增加了约 2 小时的额外工作量。]

---

## 失败案例摘要

| 案例 ID | 来源 | 根因类型 | 严重程度 | 描述 |
|--------|------|---------|---------|------|
| case-001 | PRD 评审 | A（知识缺失） | major | 缺少竞品分析章节 |
| case-002 | PRD 评审 | D（格式规范） | major | 用户故事格式不符合规范 |

---

## 根因分析

### case-001：缺少竞品分析章节

**根因类型**: A — 知识缺失

**分析**: 现有 PRD skill 的文档结构模板中没有包含竞品分析章节。AI 按照 skill 中定义的结构生成文档，因此不会主动添加这个章节。这不是 AI 的推理错误，而是 skill 中缺少这个领域知识（公司 PRD 规范要求竞品分析）。

**证据**: 评审意见原话："这份 PRD 缺少竞品分析，我们公司的 PRD 规范要求必须有这个章节"

**修复方向**: 在 skill 的文档结构模板中添加竞品分析章节，并说明该章节的内容要求。

### case-002：用户故事格式错误

**根因类型**: D — 格式/规范问题

**分析**: 现有 PRD skill 提到了用户故事，但没有指定具体的格式模板。AI 使用了通用的用户故事写法，而公司规范要求使用特定格式。

**证据**: 评审意见原话："用户故事格式不符合公司规范，应该用 'As a [role], I want [goal], so that [benefit]' 格式"

**修复方向**: 在 skill 中添加用户故事的格式模板和示例。

---

## 修改内容

### Patch 1: patch-001 — 添加竞品分析章节要求

**修复的根因**: case-001 — 类型 A（知识缺失）
**修改位置**: `## 文档结构` 章节
**修改类型**: 新增内容

**修改前**:
```
## 文档结构

PRD 应包含以下章节：
1. 背景与目标
2. 用户需求
3. 功能需求
4. 验收标准
```

**修改后**:
```
## 文档结构

PRD 应包含以下章节：
1. 背景与目标
2. 竞品分析（必须包含，分析至少 3 个竞品的相关功能）
3. 用户需求
4. 功能需求
5. 验收标准
```

**修改原因**: 公司 PRD 规范要求必须包含竞品分析章节。添加此章节要求后，AI 在生成 PRD 时会自动包含竞品分析内容，避免评审被打回。

---

### Patch 2: patch-002 — 添加用户故事格式模板

**修复的根因**: case-002 — 类型 D（格式规范）
**修改位置**: `## 用户需求` 章节
**修改类型**: 新增内容

**修改前**:
```
## 用户需求

描述目标用户的需求和痛点，使用用户故事的形式表达。
```

**修改后**:
```
## 用户需求

描述目标用户的需求和痛点，使用用户故事的形式表达。

**用户故事格式（必须遵循）**:
```
As a [用户角色], I want [目标/功能], so that [获得的价值/原因].
```

**示例**:
- As a 新注册用户, I want to complete onboarding in under 3 minutes, so that I can start using the product without frustration.
- As a 团队管理员, I want to bulk-invite team members via CSV, so that I don't have to add them one by one.
```

**修改原因**: 公司规范要求用户故事使用固定格式。添加格式模板和示例后，AI 会按照规范格式生成用户故事，避免格式不符合要求。

---

## 预期效果

修改后，AI 在生成 PRD 时应该：

1. 自动在文档结构中包含竞品分析章节，并分析至少 3 个竞品
2. 使用 "As a [role], I want [goal], so that [benefit]" 格式编写所有用户故事

---

## 验证建议

建议用以下测试用例验证修改效果：

1. 提示词："帮我生成一份关于[新功能]的 PRD" → 检查是否包含竞品分析章节
2. 提示词："写 3 个用户故事，描述[功能]的使用场景" → 检查格式是否符合规范

---

## 风险评估

**低风险**。本次修改只是在现有结构中新增内容，没有删除或修改原有逻辑。

潜在副作用：竞品分析章节的添加可能使 PRD 生成时间略有增加（AI 需要生成更多内容）。

## 使用说明

1. 将上述模板中的 `[占位符]` 替换为实际内容
2. 根据实际 Patch 数量，复制"修改内容"章节中的 Patch 块
3. 运行 `scripts/generate_diff.py` 生成 diff，填入"完整 Diff"章节
4. 保存到 `~/.catpaw/skills/[skill-name]/pr/pr-[timestamp].md`
