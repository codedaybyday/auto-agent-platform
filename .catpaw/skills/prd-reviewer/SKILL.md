# PRD 文档质量评估 Skill

评估 AI 生成的结构化需求文档质量,以原始 PRD 文档为标准,从三个维度进行评估,输出评分报告和改进建议。

> 并行评估：使用 3 个 subagent 并行执行评估，包括原始 PRD 分析、内置评估标准、业务自定义标准。

---

## 评估流程

```
第一步: 获取评估对象
第二步: 并行评估（3个subagent）
第三步: 汇总评估结果
第四步: 计算总分和判定
第五步: 生成评估报告
第六步: 询问是否自动修复
第七步: 执行自动修复（如用户确认）
```

---

### 第一步:获取评估对象

#### 1.1 确认原始 PRD

**重要**: 在开始评估前,必须先向用户确认原始 PRD 文档,确保评估基准正确。

**确认流程**:
1. 识别原始 PRD 来源（链接/文件路径/用户输入）
2. 获取原始 PRD 内容
3. 向用户展示并确认

```
📋 已获取原始 PRD 文档:

**文档标题**: [从文档中提取的标题]
**文档来源**: [链接或文件路径]
**核心内容摘要**:
- 背景: [简要描述]
- 目标: [简要描述]
- 主要功能点: [列举核心功能]

请确认: 以上是否为需要评估的原始 PRD 文档?
- 输入「是」或「y」→ 继续评估流程
- 输入「否」或「n」→ 请提供正确的原始 PRD 链接或文件
```

#### 1.2 获取结构化文档

用户确认原始 PRD 后,获取待评估的结构化文档:

```bash
# 读取用户指定的结构化文档
cat <结构化文档路径>
```

如果用户未提供结构化文档路径,按照以下优先级查找:
1. 项目 `.catpaw/docs/*/prd.md` 目录下的文件
2. 询问用户指定具体路径

---

### 第二步:并行评估（使用 subagent）

使用 Task 工具创建 3 个并行 subagent:

| Agent | 任务 |
|-------|------|
| Agent 1 | 原始 PRD 分析 |
| Agent 2 | 内置评估标准评估 |
| Agent 3 | 业务自定义标准评估 |

> 详细评估标准和扣分规则见 `references/prd-quality-criteria.md`

#### 2.1 Subagent 1: 原始 PRD 分析器

**任务**: 分析原始 PRD 文档内容

**执行步骤**:
1. 读取原始 PRD 文档完整内容
2. 提取核心信息（背景、目标、功能点、业务规则、验收标准等）
3. 识别 PRD 中存在的问题（信息缺失、表述模糊、逻辑矛盾等）

**返回**:
```json
{
  "prdSummary": { "背景、目标、功能点等核心信息" },
  "prdIssues": [{ "PRD本身的问题列表" }],
  "prdQuality": { "完整性、清晰度、一致性等评分" }
}
```

#### 2.2 Subagent 2: 内置评估标准器

**任务**: 基于内置标准评估结构化文档

**执行步骤**:
1. 读取 `references/prd-quality-criteria.md` 获取内置评估标准
2. 读取结构化文档 `prd.md`
3. 按内置标准评估三个维度

**评估维度**:
- **内容还原度 (40 分)**: 核心信息完整性、信息准确性、数据准确性、关联正确性
- **PRD 质量 (40 分)**: 完整性、清晰度、一致性、可测试性、依赖识别
- **结构化质量 (20 分)**: 功能点划分、待确认问题、UI 布局描述

**返回**:
```json
{
  "scores": { "各项评分" },
  "issues": [{ "发现的问题列表" }]
}
```

#### 2.3 Subagent 3: 业务自定义评估器

**任务**: 基于业务自定义标准评估

**执行步骤**:
1. 扫描项目 `.catpaw/skills/prd-reviewer/references/` 目录下的所有 `custom-*.md` 文件
2. 读取业务自定义评估标准
3. 按自定义标准评估结构化文档

**返回**:
```json
{
  "hasCustomRules": true,
  "customCriteria": { "自定义评估标准" },
  "customScores": { "自定义评分" },
  "customIssues": [{ "自定义问题列表" }]
}
```

---

### 第三步:汇总评估结果

汇总 3 个 subagent 的返回结果：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 评估结果汇总
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【原始 PRD 分析】
- 核心功能点: X 个
- PRD 问题: Y 个 (严重 Z 个)

【内置标准评估】
- 内容还原度: 35/40
- PRD 质量: 32/40
- 结构化质量: 16/20
- 小计: 83/100

【业务自定义评估】
- 自定义评分: 8/10
- 自定义问题: 2 个

【综合评分】: 91/110
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

### 第四步:计算总分和判定

**总分计算**:
```
总分 = 内容还原度 (40) + PRD 质量 (40) + 结构化质量 (20) + 自定义评分 (如有)
```

**判定标准**:

| 总分 | 判定结果 | 说明 |
|------|----------|------|
| 90-100+ | 通过 ✅ | 文档质量优秀,可直接用于开发 |
| 80-89 | 基本通过 ⚠️ | 有小问题,建议修复后使用 |
| 70-79 | 需改进 🔧 | 存在明显问题,需补充信息 |
| <70 | 不通过 ❌ | 严重问题,需重新解析或补充 PRD |

---

### 第五步:生成评估报告

**输出路径**:与结构化文档同目录,文件名为 `prd-review.md`

> 报告模板见 `assets/report-template.md`

写入完成后,告知用户 `prd-review.md` 的完整路径,并在对话中输出报告全文供直接阅读。

---

### 第六步:询问是否自动修复

```
评估已完成,总分 X/100+,发现 Y 个问题。

是否需要自动修复 prd.md 中的问题?
- 输入「是」或「y」→ 执行自动修复
- 输入「否」或「n」→ 结束评估流程
- 输入「仅修复严重问题」→ 只修复 🔴 必须修复的问题
```

---

### 第七步:执行自动修复

根据评估报告中发现的问题,自动修改结构化文档 `prd.md`。

**修复范围分类**:

| 问题类型 | 处理方式 |
|---------|--------|
| ❌ 缺失信息 | 从原始 PRD 中提取缺失内容并补充 |
| ⚠️ 错误信息 | 按原始 PRD 内容修正错误描述 |
| ℹ️ 冗余信息 | 删除不必要的添加内容 |
| 原始 PRD 缺少信息 | 询问用户是否标注「待补充」 |
| 业务规则不明确 | 仅在报告中标注,需人工处理 |

> 详细修复原则见 `references/fix-guidelines.md`

**修复完成输出**:

```markdown
## 修复完成

✅ 已修复 X 个问题
📝 修复内容摘要: [修复列表]
⚠️ 需人工处理的问题: N 个

📄 结构化文档已更新: {prd.md 路径}
📄 评估报告已更新: {prd-review.md 路径}
```

---

## 注意事项

1. **客观性**: 以原始 PRD 为准,避免主观判断
2. **建设性**: 问题要具体,建议要可操作
3. **实用性**: 关注影响开发的问题,不纠结细枝末节
4. **并行性**: 三个 subagent 并行执行，提高评估效率
