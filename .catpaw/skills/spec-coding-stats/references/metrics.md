# Agent Code Stats 指标说明

## 核心概念

### Agent 记录 (AI Record)

每次 Agent 编辑文件后，`after-file-edit` hook 自动调用 `addRecord`，将变更的代码片段写入缓存目录。每条记录包含：
- `file_path`: 被修改的文件路径（绝对路径）
- `change_type`: 变更类型（`新增` | `删除`）
- `change_code`: 变更的具体代码片段（多行字符串）

### 匹配策略

统计阶段通过**代码行精确匹配**判断归属：
1. 将 Agent 记录的 `change_code` 按行拆分，构建索引：`file_path -> Set<trimmed_line>`
2. 对 `git diff` 中的每行变更内容（去除 `+`/`-` 后的代码），检查是否存在于对应文件的索引中
3. 若匹配到 → 判定为 **AI 生成**；否则 → 判定为 **人工编写**

## 指标详解

| 指标 | 定义 | 计算公式 |
|------|------|---------|
| AI 新增行数 | diff 中的 `+` 行，且内容存在于 Agent 记录 | Σ(diff `+` 行 ∧ 匹配记录) |
| AI 删除行数 | diff 中的 `-` 行，且内容存在于 Agent 记录 | Σ(diff `-` 行 ∧ 匹配记录) |
| 人工新增行数 | diff 中的 `+` 行，但未匹配 Agent 记录 | Σ(diff `+` 行 ∧ 未匹配) |
| 人工删除行数 | diff 中的 `-` 行，但未匹配 Agent 记录 | Σ(diff `-` 行 ∧ 未匹配) |
| AI 新增占比 | AI 新增占所有新增的比例 | AI新增 / (AI新增 + 人工新增) × 100% |
| AI 变更占比 | AI 变更占总变更的比例 | (AI新增 + AI删除) / 总变更行数 × 100% |

### AI 新增占比参考

| 范围 | 含义 |
|------|------|
| 100% | 所有新增代码均由 AI 生成 |
| 80-100% | AI 主导新增代码编写 |
| 50-80% | AI 与人工协作均衡 |
| < 50% | 人工主导新增代码编写 |

## 代码匹配示例

Agent 编辑 `src/index.tsx`，记录了：
```json
{ "change_type": "新增", "change_code": "const foo = 1;\nconst bar = 2;\nconsole.log(foo + bar);" }
```

对应 `git diff` 输出：
```diff
+ const foo = 1;           → AI新增
+ const bar = 2;           → AI新增
+ const baz = 3;           → 人工新增
- console.log(foo + bar);  → AI删除
- const old = 0;           → 人工删除
```

结果：AI新增 2 行，人工新增 1 行，AI删除 1 行，人工删除 1 行，AI新增占比 66.7%，AI变更占比 60%。

## 文件路径匹配策略

1. **精确匹配**：diff 文件路径与记录 `file_path` 规范化后完全一致
2. **模糊匹配（后缀匹配）**：精确匹配失败时，检查路径后缀是否互相包含，处理绝对路径 vs 相对路径的差异

## 常见问题

**Q: 为什么使用代码行匹配而不是 git blame?**  
关注当前 session 的实时统计，不依赖 commit message 格式，统计粒度精确到行，且支持未提交的本地变更。

**Q: 代码修改后还能匹配吗？**  
不能。匹配基于代码行的精确内容，后续修改（格式化、重命名）会导致无法匹配，这些行将被统计为人工编写。

**Q: 空行会被统计吗？**  
不会，统计时过滤掉 trim 后为空字符串的行。
