# PR 发起（create-pr）

创建 Pull Request，支持自动从 commit 生成描述、指定评审人。

---

## 前置依赖

使用此命令需要安装 Code CLI：

```bash
npm install -g @ee/code-cli --registry=http://r.npm.sankuai.com
```

检查是否已安装：

```bash
code-cli --version
```

如果未安装或命令不存在，提示用户安装后再继续。

---

## 参数

- `$ARGUMENTS`：PR 标题（可选，不填则从最近 commit 自动生成）

---

## 第一步：收集 PR 信息

### 1.1 获取分支信息

```bash
# 获取当前分支
git branch --show-current

# 获取目标分支（默认 master 或 main）
git remote show origin | grep "HEAD branch"

# 获取最近的 commits
git log master..HEAD --oneline
```

### 1.2 生成 PR 信息

- **标题**：用户提供的标题 或 从最近 commit 提取
- **描述**：从 commits 整合 或 从变更生成

### 1.3 确认评审人

```
请指定评审人（输入用户名，多个用逗号分隔）：
评审人：{等待用户输入}
```

**⚠️ 停止执行，等待用户确认。**

---

## 第三步：调用 mep-code skill 创建 PR

调用 `/mep-code` skill 执行创建：

```bash
code-cli pr create \
  --source-branch <当前分支> \
  --target-branch master \
  --title "<标题>" \
  --description "<描述>" \
  --reviewers "<评审人>"
```

## 可选：草稿 PR

如果用户说「创建草稿 PR」「先创建一个 draft」，则添加 `--draft` 参数：

```bash
code-cli pr create --draft ...
```

---

## 错误处理

| 错误情况 | 处理方式 |
|---------|---------|
| 分支不存在 | 提示检查分支名称 |
| 无权限 | 提示联系仓库管理员 |
| 合并冲突 | 展示冲突文件，提示先解决冲突 |
| CI 未通过 | 展示失败原因，询问是否仍要创建 |
