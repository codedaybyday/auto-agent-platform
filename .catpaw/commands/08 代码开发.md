# 代码开发 Command

智能分发命令，自动识别任务类型并调用对应的开发流程。

## 使用方式
/08 代码开发 <任务ID或标题>

**示例**:
- `/08 代码开发 T05`
- `/08 代码开发 用户中心页面`

---

<workflow>
  <step id="1" name="解析任务信息" required="true">

---

## 执行规则（全局）

1. **原子化执行**：必须按 Step 1 → 2 → 3 → 4 顺序执行，禁止跳步
2. **完成确认**：每步完成后必须输出 `✓ Step X 完成：[具体说明]`
3. **等待Skill**：Step 3 必须等待Skill 返回完成确认，不假设执行成功
4. **禁止跳过**：即使Skill 看起来“简单”或“之前执行过”，也必须完整执行 Step 3
5. **失败处理**：任何步骤失败时，输出原因并停止，不继续后续步骤

---

## Skill 的完成确认格式要求

被调用的Skill（ui-driven-development / logic-driven-development）**必须**在完成后输出：
✓ {skill-name} 完成

产出文件：[文件列表]

关键变更：[变更摘要]

若Skill 未按此格式输出，Command视为执行失败。
