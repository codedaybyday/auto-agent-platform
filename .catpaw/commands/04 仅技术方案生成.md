# 技术方案生成（gen-techdoc）

根据已解析的需求文档（prd.md）和可选的原始 KM 文档，自动生成结构完整的前端技术方案，包含功能设计、影响范围分析、稳定性保障、工时评估等章节。

## 使用方式

```
/gen-techdoc <prd-file-path> [km-url]
```

示例：
- `/gen-techdoc .catpaw/docs/merchant-promo-perm/prd.md https://km.sankuai.com/collabpage/2747985261`
- `/gen-techdoc .catpaw/docs/merchant-promo-perm/prd.md`（仅基于 prd.md 生成）

## 参数说明

- `<prd-file-path>`（必填）：需求解析结果文件路径，通常位于 `.catpaw/docs/{需求英文名}/prd.md`
- `[km-url]`（可选）：原始 KM 文档链接，提供后可补充 prd.md 中未覆盖的细节

## 执行步骤

### 第一步：执行 techdoc-generator skill

执行 `/techdoc-generator` skill，严格按照 skill 中的完整流程执行。

### 第二步：询问是否触发技术方案评审

技术方案写入完成后，询问用户：「是否需要对技术方案进行评审？」

- 若用户确认：执行 `/techdoc-reviewer` skill，对刚生成的 `techdoc.md` 进行质量评审
- 若用户跳过：直接结束，告知用户可随时手动触发评审

### 错误处理

- 若 `<prd-file-path>` 文件不存在：告知用户文件路径有误，停止执行
- 若 skill 执行失败：告知用户具体错误原因，停止执行
- 若 `km` 命令不存在：先执行 `source ~/.meituan-local-tools/.venv/bin/activate`，若仍不可用则跳过原始文档读取，仅基于 `prd.md` 生成
- 若 KM 文档无法访问（权限不足）：跳过原始文档读取，仅基于 `prd.md` 生成，在文档中注明

### 成功标准

- `techdoc.md` 已写入 `{项目根目录}/.catpaw/docs/{需求英文名}/techdoc.md`
- 告知用户文件的完整路径
