# AI 编码统计上报

在开发前/后上报 AI 编码统计数据，用于记录 AI 辅助开发的效率指标。

## 参数

- `$ARGUMENTS`：上报阶段标识
  - `baseline`：开发开始前的基线统计（默认）
  - `final`：开发完成后的最终统计

---

## 执行步骤

### 第一步：检查是否已通过环境检测绑定 ONES 需求

**重要提示**：上报统计前必须先通过 `/00 环境检测` 绑定 ONES 需求，否则统计数据无法关联到具体需求。

```
❓ 请确认：
   是否已通过 /00 环境检测 绑定了 ONES 需求？
   
   如果尚未绑定，必须先执行：
     /00 环境检测
   
   按照环境检测的引导完成 ONES 需求绑定后，再继续上报统计。
```

**用户确认已绑定后继续执行**。如果用户未绑定，**停止执行**，提示用户先执行 `/00 环境检测`。

---

### 第二步：提示用户先提交代码

**重要提示**：上报统计前请确保当前代码已提交（commit），否则统计平台可能无法正确统计 AI 编码数据。

```
⚠️  上报统计前请确认：
   当前代码变更是否已提交（git commit）？
   
   如果尚未提交，建议先执行：
     git add .
     git commit -m "xxx"
   
   然后再继续上报统计。
```

**用户确认后继续执行**。

---

### 第三步：执行 ai-coding-stats 统计并提交

执行以下命令统计 AI 编码占比并提交到平台：

```bash
ai-coding-stats stats --submit
```

该命令会自动完成以下操作：
1. 统计当前分支的 AI 编码占比（新增行数、删除行数、占比等）
2. 将统计数据（包括各阶段进展、Spec Commit 指标等）一并提交到统计平台

---

### 第四步：输出结果

直接将 `ai-coding-stats stats --submit` 的输出结果展示给用户即可，同时告知统计平台链接：

```
📈 统计平台：https://api-call-tracker-hub.mynocode.host/#/requirement/${requirement_id}
```

---

## 错误处理

| 错误情况 | 处理方式 |
|---------|---------|
| 非 git 仓库 | skill 会输出错误信息并停止执行 |
| skill 未安装 | skill 会提示安装并停止执行 |
| 未绑定 ONES | 提示用户先执行 `/00 环境检测` 完成绑定 |
| 上报失败（网络/平台问题） | skill 会展示本地统计结果，提示手动重试 |
