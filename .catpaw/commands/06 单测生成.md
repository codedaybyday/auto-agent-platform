# 单元测试生成（gen-unittest）

根据技术方案文档和项目源代码，自动分析被测模块、识别测试场景，生成覆盖主流程、边界条件和异常处理的 Jest 单测文件，统一输出到 `__tests__/` 目录。

## 使用方式

```
/gen-unittest [tech-design-file-path] [source-file-path]
```

示例：
- `/gen-unittest .catpaw/docs/merchant-promo-perm/tech-design.md`（根据技术方案生成）
- `/gen-unittest .catpaw/docs/merchant-promo-perm/tech-design.md src/components/UserCard.tsx`（指定源文件）
- `/gen-unittest`（自动查找当前项目最近的 tech-design.md）

## 参数说明

- `[tech-design-file-path]`（可选）：技术方案文件路径；不提供时自动查找 `.catpaw/docs/*/tech-design.md` 中最近修改的一份
- `[source-file-path]`（可选）：指定只为某个源文件生成单测；不提供时根据技术方案中的模块清单批量生成

## 执行步骤

### 第一步：执行 unit-test-generator skill

执行 `/unit-test-generator` skill，严格按照 skill 中的完整流程执行。

### 错误处理

- 若 skill 执行失败：告知用户具体错误原因，停止执行
- 若技术方案文件不存在：告知用户路径有误，停止执行
- 若源文件不存在（模块尚未开发）：生成测试骨架，用 `// TODO` 标注待补充部分，继续执行
- 若 `__tests__/` 目录不存在：自动创建后继续

### 成功标准

- 所有测试文件已写入 `__tests__/` 对应子目录
- 在对话中输出覆盖摘要：生成文件数、测试用例数、三类场景分布
- 告知用户运行命令：`npx jest __tests__/ --coverage`
