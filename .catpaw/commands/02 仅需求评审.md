# 需求文档评审（review-prd）

以原始 KM 文档为真相来源，对 AI 生成的结构化需求文档（prd.md）进行系统性评审。**评审前会先向用户确认原始 PRD 文档**，确保评估基准正确，然后识别内容遗漏、失真、AI 幻觉及 PRD 质量问题，输出带评分和改进建议的评审报告。

## 使用方式

```
/review-prd <prd-file-path> [km-url]
```

示例：
- `/review-prd .catpaw/docs/merchant-promo-perm/prd.md https://km.sankuai.com/collabpage/2747985261`
- `/review-prd .catpaw/docs/merchant-promo-perm/prd.md`（仅做 PRD 质量评审，无还原度对比）

## 参数说明

- `<prd-file-path>`（必填）：待评审的 prd.md 文件路径，通常位于 `.catpaw/docs/{需求英文名}/prd.md`
- `[km-url]`（可选）：原始 KM 文档链接，提供后可同时进行还原度评审；不提供则仅做 PRD 质量评审

## 执行步骤

### 第一步：执行 prd-reviewer skill

执行 `/prd-reviewer` skill，严格按照 skill 中的完整流程执行。

### 第二步：询问是否自动修复问题

评审结果写入完成后，若评审报告中存在问题，询问用户：「是否需要自动修复发现的问题？」

- 若用户确认：根据评审报告中的改进建议，自动修改 `prd.md` 文件
- 若用户跳过：直接结束，告知用户可根据评审报告手动修复问题

### 错误处理

- 若 `<prd-file-path>` 文件不存在：告知用户文件路径有误，停止执行
- 若 `km` 命令不存在：先执行 `source ~/.meituan-local-tools/.venv/bin/activate`，若仍不可用则提示用户参考 km-doc-tools skill 安装说明
- 若 KM 文档无法访问（权限不足）：跳过还原度评审，仅做 PRD 质量评审，在报告中注明原因

### 成功标准

- `prd-review.md` 已写入与 `prd.md` 同级目录
- 告知用户 `prd-review.md` 的完整路径
- 在对话中输出报告全文供直接阅读
