# PRD 文档自动解析与修复

从学城 KM 文档 URL 自动执行完整的需求文档解析流程：解析 → 验证 → 修复 → 再验证，输出高质量的结构化 PRD 文档。

## What This Command Does

1. **解析需求文档**: 执行 prd-parser skill 从 KM 文档提取结构化需求内容
2. **质量验证**: 执行 prd-reviewer skill 对生成的文档进行三维度评估
3. **问题修复**: 根据评估报告中的问题清单自动修复 prd.md
4. **循环优化**: 修复后重新验证，直到文档质量达标或达到最大迭代次数

## Usage

```bash
# 使用 KM 文档 URL
/parse-prd-auto https://km.sankuai.com/collabpage/2747985261

# 使用 KM 文档 ID
/parse-prd-auto 2747985261
```

## Implementation Steps

When this command is invoked:

### 1. 解析参数并启动流程

**解析用户输入**:

- 如果输入是完整 URL (如 `https://km.sankuai.com/collabpage/2747985261`)，提取 doc_id
- 如果输入是纯数字，直接作为 doc_id 使用

**创建 Todo 列表**:

```
1. 解析 KM 需求文档
2. 验证文档质量
3. 修复问题并优化
4. 最终确认
```

**记录项目根目录**: 使用 `pwd` 获取当前项目路径

---

### 2. 调用 prd-parser Skill 进行解析

**执行 skill**: 执行 `/prd-parser` skill，严格按照 skill 中的完整流程执行。

**标记第一个 todo 为完成**

**输出示例**:
```
✅ 需求解析完成！

📄 结构化文档: .catpaw/docs/fake-dish-min-order/prd.md
```

---

### 3. 调用 prd-reviewer Skill 进行质量验证

**执行 skill**: 执行 `/prd-reviewer` skill，严格按照 skill 中的完整流程执行。

**标记第二个 todo 为完成**

---

### 4. 分析评估报告并修复问题

**读取评估报告**: 读取刚生成的 `prd-review.md`

**提取问题清单**:

从评估报告中提取三类问题:
- **必须修复 (阻塞开发)**: 严重影响开发的问题
- **建议修复 (影响体验)**: 影响理解但不阻塞的问题
- **可选优化 (锦上添花)**: 锦上添花的建议

**优先级处理**:
1. 首先处理「必须修复」问题
2. 其次处理「建议修复」问题
3. 最后处理「可选优化」(如果评分 < 90)

**修复方法**:

针对常见问题类型执行相应修复:

| 问题类型 | 修复操作 |
|---------|---------|
| 核心功能点遗漏 | 从原始 KM 文档补充遗漏的功能点 |
| 验收标准模糊 | 具体化验收条件，添加可测试的断言 |
| 接口依赖缺失 | 补充接口说明章节 |
| UI 描述不清晰 | 重新解析原型图，补充详细布局描述 |
| 待确认问题无效 | 移除无效问题，或从原文找到答案 |
| 业务规则错误 | 对照原始文档修正规则描述 |

**修复执行**:
- 使用 Read tool 读取当前 prd.md
- 使用 Edit tool 进行精准修改
- 每修复一个问题，输出修复日志

**标记第三个 todo 为完成**

---

### 5. 循环验证 (迭代修复)

**最大迭代次数**: 3 次

**迭代逻辑**:

```
iteration = 1
while iteration <= 3:
    重新执行 prd-reviewer 验证修复后的文档
    读取新的评估报告
    
    if 评分 >= 80:
        break  # 质量达标，退出循环
    
    if 评分没有提升 or 问题数量没有减少:
        break  # 无法继续优化，退出循环
    
    iteration += 1
    继续修复剩余问题
```

**停止条件**:
- 评分达到 80+ (基本通过)
- 达到最大迭代次数 (3次)
- 评分连续两次无提升

---

### 6. 最终报告

**输出最终结果**:

```

**原始 KM 文档**: https://km.sankuai.com/collabpage/{doc_id}

**输出文件**:
- 结构化文档: .catpaw/docs/{需求英文名}/prd.md
- 评估报告: .catpaw/docs/{需求英文名}/prd-review.md

**最终评分**: X/100 (通过/基本通过/需改进)

**迭代情况**:
- 第 1 次解析评分: X/100
- 第 N 次修复后评分: X/100

**已修复问题**:
1. [问题描述]
2. [问题描述]

**剩余问题** (如有):
1. [问题描述] - 需要人工确认/补充

**下一步建议**:
- [根据最终评分给出建议]
```

**标记所有 todo 为完成**

## Important Notes

- **执行 skill**: 在执行解析和验证时，执行对应的 skill
- **遵循 skill 的输出模板**: 使用 `references/output-template.md` 规范输出格式
- **每次修复后都要重新验证**: 确保修复没有引入新问题
- **保留原始评估报告**: 每次迭代的评估报告追加时间戳保存，便于对比
- **不要捏造内容**: 修复时只能从原始 KM 文档补充信息，不能凭空捏造

## Error Handling

如果任何步骤失败:

1. **km CLI 不可用**:
   - 输出错误信息: "km CLI 工具未安装，请参考 km-doc-tools skill 安装说明"
   - 停止执行

2. **KM 文档获取失败**:
   - 输出错误信息: "无法获取 KM 文档，请检查链接或权限"
   - 停止执行

3. **原型图解析失败**:
   - 在文档中标注 "原型图链接无法访问"
   - 继续执行其他解析步骤

4. **修复失败**:
   - 输出 "自动修复失败，需要人工介入"
   - 报告当前状态和剩余问题
   - 提供原始 KM 文档链接供人工查阅
