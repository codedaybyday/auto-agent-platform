# 技术方案评审（review-techdoc）

以 PRD 需求文档、接口文档、项目知识库、实际代码和技术栈知识库为五重基准，对 AI 生成的技术设计文档进行系统性评审，识别需求覆盖缺失、接口设计出入、方案与项目现实不符、技术栈使用不规范等问题，输出带评分和改进建议的评审报告。

## 使用方式

```
/review-techdoc <techdoc-file-path> [prd-file-path] [km-url]
```

示例：
- `/review-techdoc .catpaw/docs/merchant-promo-perm/techdoc.md .catpaw/docs/merchant-promo-perm/prd.md https://km.sankuai.com/collabpage/2747985261`
- `/review-techdoc .catpaw/docs/merchant-promo-perm/techdoc.md .catpaw/docs/merchant-promo-perm/prd.md`
- `/review-techdoc .catpaw/docs/merchant-promo-perm/techdoc.md`（自动查找同目录 prd.md）

## 参数说明

- `<techdoc-file-path>`（必填）：待评审的技术方案文件路径，通常位于 `.catpaw/docs/{需求英文名}/techdoc.md`
- `[prd-file-path]`（可选）：需求文档路径；不提供时自动查找与 `techdoc` 同目录的 `prd.md`
- `[km-url]`（可选）：原始 KM 需求文档链接，提供后可补充 prd.md 中未覆盖的需求细节

## 执行步骤

### 第一步：执行 techdoc-reviewer skill

执行 `/techdoc-reviewer` skill，严格按照 skill 中的完整流程执行。

### 第二步：询问是否自动修复

评审报告生成后，询问用户：「是否需要自动修复技术方案中发现的问题？」

- 若用户确认（全部修复）：根据评审报告中的改进建议，修复所有可自动修复的问题
- 若用户选择「仅修复严重问题」：只修复 🔴 严重问题
- 若用户跳过：直接结束，告知用户可随时手动触发修复

### 错误处理

- 若 `<techdoc-file-path>` 文件不存在：告知用户文件路径有误，停止执行
- 若 `prd.md` 未提供且同目录也不存在：在报告中注明「缺少需求文档，跳过需求覆盖度评审」，继续其他维度评审
- 若 `km` 命令不存在：先执行 `source ~/.meituan-local-tools/.venv/bin/activate`，若仍不可用则跳过原始文档读取
- 若接口文档未提供：在报告中注明「缺少接口文档，跳过接口一致性评审」，继续其他维度评审

### 成功标准

- `techdoc-review.md` 已写入与 `techdoc.md` 同级目录
- 告知用户 `techdoc-review.md` 的完整路径
- 在对话中输出报告全文供直接阅读
