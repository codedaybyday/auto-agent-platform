# 知识库同步上传

将当前项目 `.catpaw` 目录下的所有文件，通过 knowledge-sync MCP 执行 `knowledge-sync sync` 同步上传到 [Block-AI 知识库管理平台](https://block.sankuai.com/ai-market/m-hub?tab=template-config&templateId=37)。

## 前置检查

### 1. 确认knowledge-sync MCP 已配置

检查当前环境是否已配置 `knowledge-sync` MCP。如果未配置，提示用户在 CatPaw MCP 设置中添加以下配置后重启 CatPaw：

```json
"knowledge-sync": {
  "command": "npx",
  "args": ["-y", "@block/memory-hub@latest"],
  "env": {
    "TEMPLATE_ID": "37"
  }
}
```

> 注意：需要 Node.js >= 18，且 npm 源必须指向美团内网：
> ```bash
> npm config set registry http://r.npm.sankuai.com/
> ```

### 2. 确认 .catpaw 目录存在

```bash
ls .catpaw/
```

如果目录不存在，提示用户先执行 `spec-workflow init` 初始化项目，再运行本命令。

## 执行步骤

### Step 1: 列出待同步文件

展示 `.catpaw` 目录下的所有文件，供用户确认同步范围：

```bash
find .catpaw -type f -not -path "*/node_modules/*" | sort
```

### Step 2: 执行 knowledge-sync sync

通过 knowledge-sync MCP，以 `.catpaw` 目录作为同步根路径执行上传：

**无参数（单工程）：**
```
knowledge-sync sync
```

**传入 `all` 参数（多工程 monorepo）：**
```
knowledge-sync sync all
```

等待同步完成，观察输出结果：
- 成功 → 进入 Step 3
- 失败 → 进入错误处理

### Step 3: 输出结果报告

```
## 知识库同步结果

**状态**: SUCCESS / FAILED

**项目**: [package.json 中的 name]
**同步目录**: .catpaw/
**同步模式**: 单工程 / 多工程(all)

---

### 查看同步结果

请访问 Block-AI 知识库管理平台：
https://block.sankuai.com/ai-market/m-hub?tab=memory-repo

在页面中输入仓库名称进行搜索，即可查看对应分支的知识库内容。
```

## 错误处理

**MCP 无法调用 / npx 执行失败：**
- 检查 Node.js 版本：`node -v`（需 >= 18）
- 检查 npm 源：`npm config get registry`（需指向 http://r.npm.sankuai.com/）
- 清除 npx 缓存后重试：`npx clear-npx-cache`
- 重启 CatPaw 后重试

**.catpaw 目录不存在：**
- 先执行 `spec-workflow init` 初始化项目，再重新运行本命令
