# UI 设计稿确认 Command

在开发前确认是否有 UI 设计稿，并根据设计稿来源选择对应的解析方式。

## 使用方式

```
/UI设计稿确认
```

## 执行流程

### Step 1：询问是否有 UI 设计稿

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎨 UI 设计稿确认
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
本次需求是否涉及 UI 设计？

1. 有 UI 设计稿（需要解析设计稿样式）
2. 无 UI 设计稿（纯逻辑/API 开发）

请输入您的选择（1 或 2）：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 2：如有设计稿，询问设计稿来源

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 请选择设计稿来源
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
您的 UI 设计稿来自哪个平台？

1. 印迹设计稿（ingee.meituan.com）
2. MasterGo 设计稿（imd.sankuai.com）

请输入您的选择（1 或 2）：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 3：根据选择提供链接示例和输入提示

#### 选项 1：印迹设计稿

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📐 印迹设计稿
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
请提供印迹设计稿信息：

**链接格式示例**：
- 项目链接：https://ingee.meituan.com/#/detail/212785
- 单个设计稿：https://ingee.meituan.com/#/artboard/123456

**输入方式**：
1. 提供 Project ID（如：212785）→ 批量解析项目内所有设计稿
2. 提供完整项目链接 → 批量解析项目内所有设计稿
3. 提供单个设计稿链接 → 仅解析该设计稿

请输入您的印迹设计稿 Project ID 或链接：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 选项 2：MasterGo 设计稿

**⚠️ 重要提示**：MasterGo 设计稿解析需要通过 CatPaw Desk 获取详细设计信息。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📐 MasterGo 设计稿
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚠️ MasterGo 设计稿解析需要 CatPaw Desk 支持

请先确认 CatPaw Desk 是否已打开：
- ✅ 已打开 CatPaw Desk
- ❌ 未打开 → 请先打开 CatPaw Desk 后再继续

CatPaw Desk 可通过以下方式打开：
- macOS：菜单栏 CatPaw 图标 → 打开 CatPaw Desk

确认 CatPaw Desk 已打开后，请回复「已打开」继续
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**用户确认 CatPaw Desk 已打开后**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📐 MasterGo 设计稿（续）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
请提供 MasterGo 设计稿信息：

**链接格式示例**：
- 完整链接：https://imd.sankuai.com/file/137265346758957?page_id=10:234
- 短链接：https://imd.sankuai.com/goto/RDNFo8LN

**输入方式**：
1. 提供完整文件链接 → 获取文件内所有页面模块
2. 提供短链接 → 自动解析到真实链接后获取模块

请输入您的 MasterGo 设计稿链接：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 4：确认需求目录

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📁 需求目录确认
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

已检测到需求目录：.catpaw/docs/{需求名}/

是否使用此目录？
1. 是，使用此目录
2. 否，我要指定其他目录

请输入您的选择：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 5：调用对应的 Skill

根据用户选择，调用对应的 skill：

| 设计稿来源 | Skill | 说明 |
|------------|-------|------|
| 印迹 - Project ID | `ingee-batch-analyzer` | 批量解析项目内所有设计稿 |
| 印迹 - 单个设计稿 | `ui-design-analyzer` | 解析单个设计稿 |
| MasterGo | `mastergo-batch-analyzer` | 批量解析 MasterGo 文件 |

```bash
# 印迹项目批量解析
.catpaw/scripts/invoke-skill.sh ingee-batch-analyzer

# 印迹单个设计稿解析
.catpaw/scripts/invoke-skill.sh ui-design-analyzer

# MasterGo 批量解析
.catpaw/scripts/invoke-skill.sh mastergo-batch-analyzer
```

## 注意事项

1. **印迹设计稿**：需要印迹访问权限
2. **MasterGo 设计稿**：
   - 需要已登录 imd.sankuai.com
   - **必须打开 CatPaw Desk** 才能获取详细的设计稿信息（节点结构、样式属性等）
   - 未打开 CatPaw Desk 时只能获取基础信息，无法获取完整设计细节
3. **需求目录**：优先使用已存在的 techdoc.md 所在目录
4. **批量解析**：设计稿数量较多时会分批并行处理
