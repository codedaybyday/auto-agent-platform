# 需求文档解析（parse-prd）

从学城 KM 文档中读取 PRD/需求文档，按标准模板输出结构化需求分析报告。

## 使用方式

```
/parse-prd <km-url>
```

示例：
- `/parse-prd https://km.sankuai.com/collabpage/2747985261`
- `/parse-prd https://km.sankuai.com/page/12345678`

## 执行步骤

### 第一步：执行 prd-parser skill

执行 `/prd-parser` skill，严格按照 skill 中的完整流程执行

### 第二步：询问是否触发需求文档评审

解析结果写入完成后，询问用户：「是否需要对解析结果进行质量评审？」

- 若用户确认：执行 `/prd-reviewer` skill，对刚生成的 `prd.md` 进行质量评审
  - 评审所需材料：原始 KM 文档（`<km-url>`）+ 刚写入的 `prd.md` 文件路径
- 若用户跳过：直接结束，告知用户可随时手动触发评审

### 错误处理

- 若 `km` 命令不存在：先执行 `source ~/.meituan-local-tools/.venv/bin/activate`，若仍不可用则提示用户参考 km-doc-tools skill 安装说明
- 若文档无法访问（权限不足）：告知用户并停止
- 若原型图链接失效：在报告中注明「原型图链接无法访问」，继续解析其他内容

### 成功标准

- `prd.md` 文件已写入 `{项目根目录}/.catpaw/docs/{需求英文名}/prd.md`
- 告知用户文件的完整路径
