# 文档编写规则

本规则在创建或修改 `*.md` 文件时自动生效。

## 文档分类原则

所有文档必须按功能模块分类存放，禁止散落在根目录：

```
docs/
├── agent/          # Agent Loop、框架设计
├── architecture/   # 系统架构、核心组件
├── bash/           # 命令行工具
├── browser/        # 浏览器自动化
├── development/    # 开发指南、问题记录
├── mcp/            # MCP 协议相关
├── performance/    # 性能分析、优化
├── reports/        # 项目报告、总结
├── session/        # 会话管理
└── storage/        # 存储方案
```

### 禁止放在根目录的文档类型

- 性能分析报告
- 设计文档
- 会议纪要
- 临时笔记

### 可以保留在根目录

- `README.md` - 项目主文档
- `CLAUDE.md` - Claude Code 配置
- `LICENSE` - 许可证
- `CHANGELOG.md` - 变更日志

## 文档命名规范

- 使用 **kebab-case**（短横线连接）
- 必须包含英文关键词，方便搜索
- 避免使用中文文件名

### 示例

```
✅ 正确: agent-loop-concurrency.md, browser-sso-cdp-solution.md
❌ 错误: 代理循环设计.md, browser/CDP方案.md
```

## 文档头部模板

每个文档必须包含以下头部信息：

```markdown
# 文档标题

## 背景
简要说明为什么要写这个文档

## 目标
文档要解决什么问题

## 内容
...

## 结论/建议
...

## 相关文档
- [链接1](路径)
- [链接2](路径)

## 变更记录
| 日期 | 作者 | 变更内容 |
|------|------|----------|
| 2024-01-01 | 张三 | 创建文档 |
```

## 文档内容规范

- **简洁性**：避免冗长，重点突出
- **结构化**：使用标题层级（# ## ###）
- **代码块**：必须标注语言类型
- **图表**：复杂流程使用 Mermaid 或 ASCII

## 文档维护

- 定期清理过期文档（每季度 review 一次）
- 文档移动时必须更新引用链接
- 新增文档后必须在 `docs/README.md` 索引中添加条目
