# /analyze-root-cause - 强制根因分析命令

**重要：在你进行任何代码修改前，必须先使用此命令！**

## 执行步骤（严格遵守）

### 第1步：收集信息
- 查看完整的错误堆栈（包括行号和调用链）
- 确认错误发生的触发条件（特定输入？特定环境？）
- 检查最近的相关代码变更

### 第2步：根因分析（禁止猜测）
输出以下格式的分析报告：

#### 表面现象
[一句话描述用户看到的问题]

#### 根本原因（至少问3次“为什么”）
为什么发生错误？
→ [答案] 为什么会有这个答案？
→ [答案] 为什么这个答案成立？
→ [最终根因]

#### 证据链
- 证据1：代码中的具体位置（行号+代码片段）
- 证据2：数据流（变量在何处赋值、何处被使用）
- 证据3：外部因素（API 返回格式、数据库状态等）

#### 影响范围
- 直接影响：[哪些功能会出错]
- 间接影响：[可能的连锁反应]

### 第3步：验证计划
在修改前，先说明如何验证根因分析正确：
- [ ] 复现步骤（确保能稳定重现）
- [ ] 添加临时的诊断日志（如需要）
- [ ] 测试边界条件

### 第4步：获得批准
输出上述内容后，**等待用户确认**，格式：
> ✅ 根因分析完成，是否允许我进行修改？

**用户回复“允许”或“同意”之前，绝对不能执行 Edit 或 Bash 命令。**
